# PPRM File Editor

The Configuration Manager now includes a specialized editor for PPRM (Parameter) files that allows you to edit them as JSON arrays while automatically converting them back to the proper PPRM format when saving.

## What are PPRM Files?

PPRM files use a simple text format where each line defines a parameter value:
- **Array format**: `VarName[Index]=Value`
- **Scalar format**: `VarName=Value`

### Examples:
```
# Motor control parameters
Motor.Speed[0]=100
Motor.Speed[1]=200
Motor.Speed[2]=300
Config.Enabled=true
AX1.KP[1]=7.615386
AX1.KI[1]=1310.14514
```

## Using the PPRM Editor

### 1. Opening the Editor
- Navigate to any configuration that contains PPRM files
- Find the PPRM file in the file management panel
- Click the **"Edit"** button (purple button with pencil icon)
- The editor will load and parse the PPRM content into editable JSON arrays

### 2. Editor Interface

The PPRM editor provides a clean, organized interface:

#### **Left Panel - Variables List**
- Shows all variables found in the PPRM file
- Displays the number of values for each variable
- **Add Variable**: Enter a name and click the "+" button
- **Delete Variable**: Click the trash icon next to any variable

#### **Right Panel - Variable Editor**
- Shows all values for the selected variable as an array
- Each value has an index number `[0]`, `[1]`, etc.
- **Add Value**: Click "Add Value" button
- **Remove Value**: Click trash icon next to any value
- **Edit Value**: Click in any input field to modify the value

#### **Header Information**
- Shows filename and statistics (number of variables and total values)
- Format reminder: `VarName[Index]=Value`

### 3. Editing Variables

#### **Variable Names**
- Must start with a letter or underscore
- Can contain letters, numbers, underscores, and dots
- Examples: `Motor`, `AX1.KP`, `Config.Name`

#### **Value Types**
The editor automatically detects and preserves value types:
- **Numbers**: `123`, `7.615386`, `9.27e-06`
- **Strings**: `"Primary"`, `"Secondary"`
- **Booleans**: `true`, `false`

#### **Array Indices**
- Arrays are one-based: `[1]`, `[2]`, `[3]`, etc. (index 0 does not exist)
- Missing indices are preserved as empty slots
- You can have sparse arrays (e.g., values at indices 1, 3, 6 but not 2, 4, 5)
- Empty values are automatically excluded when saving

### 4. Saving Changes

1. Make your edits in the JSON array format
2. Click **"Save PPRM File"** (blue button with checkmark)
3. The editor validates the data and converts it back to PPRM format
4. The file is saved to your storage (S3 or embedded storage)
5. The configuration is updated with the new file

## PPRM Format Details

### **Supported Line Formats**
```
# Comments (ignored)
VarName=Value                    # Scalar value (single value without index)
VarName[Index]=Value            # Array value (Index starts from 1)
Variable.Name[Index]=Value      # Variables can contain dots
```

### **Value Formats**
```
Motor.Speed[1]=100              # Number
Config.Name[1]="Primary"        # String (with quotes)
Config.Name[2]=Secondary        # String (without quotes)
System.Enabled[1]=true          # Boolean
Sensor.Calibration[5]=3.14159   # Float
Advanced.Param[10]=1.23e-06     # Scientific notation
```

### **Generated Output**
When you save, the editor generates a clean PPRM file:

```
# PPRM File - Generated by Configuration Manager
# Generated on: 2024-01-15T10:30:00.000Z

# Variable: Config.Enabled
Config.Enabled=true

# Variable: Config.Name
Config.Name[1]="Primary"
Config.Name[2]="Secondary"

# Variable: Motor.Speed
Motor.Speed[1]=100
Motor.Speed[2]=200
Motor.Speed[3]=300
```

## Error Handling

### **Validation Errors**
The editor validates your data before saving:
- Invalid variable names (must start with letter/underscore)
- Malformed PPRM format
- Missing required fields

### **Common Issues**
1. **Variable Name Rules**: Variable names must follow the pattern `[a-zA-Z_][a-zA-Z0-9_.]*`
2. **Empty Variables**: Variables must have at least one value
3. **Index Gaps**: Large index gaps are allowed but may create very long arrays

## Tips for Effective Use

### **Best Practices**
1. **Organize Variables**: Group related parameters with prefixes (e.g., `Motor.*`, `Sensor.*`)
2. **Use Descriptive Names**: Make variable names clear and meaningful
3. **Maintain Consistency**: Use consistent naming conventions across your PPRM files
4. **Document Changes**: The editor adds generation timestamps to help track changes

### **Performance Considerations**
- Large PPRM files (1000+ parameters) may take a moment to load
- Sparse arrays with very high indices (e.g., `Variable[10000]=value`) create large empty arrays
- Keep indices reasonable for better performance

## Integration with Configuration System

### **File Storage**
- PPRM files are stored in your configured storage system (S3 or embedded)
- Changes are saved atomically - either the entire file saves or it fails
- Original file is replaced when you save changes

### **Configuration Updates**
- When you save a PPRM file, the configuration is automatically updated
- File metadata (size, modification time) is refreshed
- Download links are regenerated for the new file

### **Backup and Safety**
- The system maintains backup copies during file replacement
- You can always download the original PPRM file before making changes
- Changes are validated before saving to prevent corruption

## Troubleshooting

### **Editor Won't Open**
- Ensure the file has a `.pprm` extension
- Check that you have edit permissions on the configuration
- Verify the file is not corrupted

### **Save Failures**
- Check your network connection
- Verify you have write permissions to the storage system
- Ensure the PPRM data passes validation

### **Data Loss Prevention**
- Always download a backup copy before making major changes
- Use the configuration version control features
- Test changes on a copy first for critical parameters

---

The PPRM editor makes it easy to work with parameter files while maintaining the exact format required by your systems. The JSON array interface provides a familiar editing experience while ensuring the output remains compatible with PPRM file specifications.
